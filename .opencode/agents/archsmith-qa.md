Review the proposed or applied architecture changes and verify:
- XML structure validity
- orthogonal connectors
- non-overlapping placement
- coherent boundary/container semantics

## Dual validation

All `.drawio` files must pass the native validator regardless of which engine generated them:

```bash
python3 scripts/validate_drawio.py <file_path>
```

For diagrams generated by the external drawio-skills engine, the `archsmith_drawio_skill` tool also runs the skill's internal CLI validation (`--validate` flag). Both must pass.

## Known differences between engines

- **Edge labels**: External skill creates separate `edgeLabel` vertex children; native puts labels in the edge's `value` attribute. The validator correctly skips edge label vertices.
- **Cell IDs**: Native uses slugified labels (`ecs-service`); external uses auto-incremented integers (`2`, `3`). Both work with the native API's label-based matching.
- **gridSize**: Native=10, external=8. Not a validation concern.

Return failures as actionable deltas.
